### Q1: 關於 AI，下列敘述何者正確？

- (A) AI 僅能處理結構化數據的分析
- <span style='color: #0066cc; font-weight: bold;'>(B) AI 涵蓋多種專業領域與技術</span>
- (C) AI 系統只能在學術研究中應用
- (D) AI 無法應用於金融領域

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) AI 涵蓋多種專業領域與技術</span>  
<span style='color: #646464;'>💡 為什麼？ 人工智慧 (AI) 是一門交叉學科，包含機器學習、自然語言處理、電腦視覺、機器人學等。它不僅能處理文字/數值等結構化資料，也能處理影像/語音等非結構化資料，且已廣泛應用於金融、醫療、製造等實務場景。</span>

### Q2: 下列何者最適合訓練電腦下圍棋 、 自動駕駛等動態重複地互動的問題？

- (A) 監督式學習（Supervised Learning）
- (B) 非監督式學習（Unsupervised Learning）
- (C) 半監督式學習（Semi-supervised Learning）
- <span style='color: #0066cc; font-weight: bold;'>(D) 強化學習（Reinforcement Learning）</span>

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(D) 強化學習（Reinforcement Learning）</span>  
<span style='color: #646464;'>💡 為什麼？ 強化學習透過「代理人 (Agent)」與「環境 (Environment)」的互動，並藉由獲得「獎勵 (Reward)」或「懲罰 (Penalty)」來學習行為準則。下棋與駕駛皆需根據當下狀態做連續決策，是強化學習的典型應用。</span>

### Q3: 深度學習模型中 ， 下列哪一項通常用來降低過擬合（Overfitting）問題？

- (A) 增加訓練數據量
- (B) 增加模型的複雜度
- (C) 增加學習率
- <span style='color: #0066cc; font-weight: bold;'>(D) 增加正則化項</span>

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(D) 增加正則化項</span>  
<span style='color: #646464;'>💡 為什麼？ 正則化 (Regularization，如 L1、L2) 透過在損失函數加入懲罰項，限制模型的參數權重不要過大，避免模型過度擬合訓練集中的雜訊，從而提升對新資料的泛化能力。(A) 也是正確方法，但在選項中，(D) 是技術手段上的標準做法。</span>

### Q4: 下列哪一項敘述符合 AI 治理的核心原則？

- <span style='color: #0066cc; font-weight: bold;'>(A) AI 系統的風險等級應在設計階段預先評估，並於部署後持續監控；</span>
- (B) 只要 AI 模型輸出結果準確，是否公平或透明並非治理重點；
- (C) AI 治理應聚焦在企業內部自行負責，不需外部規範參與；
- (D) 對於低風險 AI 系統皆可免除透明性與問責機制之要求

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(A) AI 系統的風險等級應在設計階段預先評估，並於部署後持續監控；</span>  
<span style='color: #646464;'>💡 為什麼？ 現代 AI 治理強調「風險基礎 (Risk-based)」與「全生命週期管理」。即在研發初期就落實隱私審查與風險分類，並在上線後持續追蹤其決策是否產生偏差或安全隱患。</span>

### Q5: 若要設計一個能夠辨識並過濾垃圾郵件的系統，應該選擇下列哪一種機器學習演算法以實現最佳效果？

- <span style='color: #0066cc; font-weight: bold;'>(A) 監督式學習（Supervised Learning）</span>
- (B) 非監督式學習（Unsupervised Learning）
- (C) 半監督式學習（Semi-supervised Learning）
- (D) 強化學習（Reinforcement Learning）

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(A) 監督式學習（Supervised Learning）</span>  
<span style='color: #646464;'>💡 為什麼？ 垃圾郵件過濾屬於「分類 (Classification)」任務。我們需要一組「已經標記好是或不是垃圾郵件」的歷史資料來訓練電腦，這種有明確答案(Label)引導的過程即為監督式學習。</span>

### Q6: 假設某國正在考慮使用 AI 技術來進行社會信用評分，並根據年齡、缺 陷、種族等特徵來評定個人的信用；同時，該國計劃在公眾場所使用遠 程生物辨識系統進行執法 ， 目的在於提高社會秩序 and 安全 。 上述AI 應用 可能會引發對隱私和個人權利的重大關注，根據歐盟《人工智慧法》 （Artificial Intelligence Act, AIA） 的風險分級，這類應用屬於哪一風險 等級？

- <span style='color: #0066cc; font-weight: bold;'>(A) 不可接受風險</span>
- (B) 高風險
- (C) 有限風險
- (D) 小或低風險

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(A) 不可接受風險</span>  
<span style='color: #646464;'>💡 為什麼？ 歐盟 AI 法案將嚴重威脅人性尊嚴、侵害人權、或對特定社群進行無差別監控的系統（如政府主導的社會信用體系、公共場所即時生物辨識）列為「不可接受風險 (Unacceptable Risk)」，通常直接禁止使用。</span>

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 1 和 2 都是對的</span>

### Q8: 下列何者不屬於深度學習模型？

- (A) 生成對抗網路 (GAN)
- (B) 循環神經網路 (RNN)
- (C) 卷積神經網路 (CNN)
- <span style='color: #0066cc; font-weight: bold;'>(D) 決策樹 (Decision Tree)</span>

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(D) 決策樹 (Decision Tree)</span>  
<span style='color: #646464;'>💡 為什麼？ 深度學習的核心特徵是「神經網路」。GAN、RNN、CNN 都是基於神經網路的高階技術；而決策樹是一種傳統的符號邏輯分支演算法，不具備神經網路的多層結構。</span>

### Q9: 數據分析中使用的決策樹模型，其主要優點為何？

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(A) 直觀易懂，具有良好的解釋性</span>  
<span style='color: #646464;'>💡 為什麼？ 決策樹以樹狀結構展現決策邏輯，每一層的分支都很明確 (例如：收入 > 5 萬嗎？)，因此其結果很容易向利害關係人解釋，這與「黑盒」的神經網路不同。</span>

### Q11: 關於「過擬合（Overfitting）」，下列敘述何者正確？

- (A) 模型在訓練數據與測試數據平分秋色
- (B) 模型在訓練數據上表現差，在測試數據上表現好
- <span style='color: #0066cc; font-weight: bold;'>(C) 模型過度配對訓練數據，導致測試新數據時效果不佳</span>
- (D) 以上皆非

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 模型過度配對訓練數據，導致測試新數據時效果不佳</span>  
<span style='color: #646464;'>💡 為什麼？ 過擬合是指模型過於「背誦」訓練集中的細節與雜訊，失去了對一般情況的判斷力 (泛化能力)。這通常發生在模型太過複雜或訓練次數過多時。</span>

### Q12: 模型在訓練數據上表現良好，但在測試數據上表現不佳。這是什麼現象？

- (A) 模型性能極佳
- (B) 模型正擬合
- <span style='color: #0066cc; font-weight: bold;'>(C) 模型過度擬合（Overfitting）</span>
- (D) 模型欠擬合（Underfitting）

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 模型過度擬合（Overfitting）</span>  
<span style='color: #646464;'>💡 為什麼？ 欠擬合 (Underfitting) 是連在訓練集都跑不好；過度擬合 (Overfitting) 則是只有訓練集跑得好，一遇到外面的新資料就破功。</span>

### Q13: 文字生成模型的產出，主要基於機率分佈預測「下一個字」是什麼，這稱為？

- (A) 單字配對
- (B) 隨機選擇
- <span style='color: #0066cc; font-weight: bold;'>(C) 自回歸生成模型 (Autoregressive Model)</span>
- (D) 排列組合

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 自回歸生成模型 (Autoregressive Model)</span>  
<span style='color: #646464;'>💡 為什麼？ LLM (如 GPT) 是自回歸模型的一種，它會根據已經生成的內容作為輸入，計算出一組機率向量，從中挑選（採樣）機率最大的作下一個字，週而復始，直到生成完整內容。</span>

### Q14: 關於大型語言模型 (LLM)，下列何者正確？

- (A) 它可以完全取代各領域專家
- <span style='color: #0066cc; font-weight: bold;'>(B) 它具備強大的文字生成與理解能力</span>
- (C) 它所生成的資訊絕對正確
- (D) 它不具備任何語意理解能力

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) 它具備強大的文字生成與理解能力</span>  
<span style='color: #646464;'>💡 為什麼？ LLM 雖然能產生豐富且具流暢性的文字，但並非「全能上帝」，它仍會產生幻覺 (Hallucination) 並生成錯誤資訊，不可完全盲信，但其在語意總結與跨領域寫作上極其優越。</span>

### Q15: 下列哪一項技術是讓機器具備如人類般理解自然語言 (NLP) 的關鍵？

- (A) 提高電腦的 CPU 時脈
- <span style='color: #0066cc; font-weight: bold;'>(B) 注意力機制 (Attention Mechanism)</span>
- (C) 增加數據的儲存容量
- (D) 採用更多 GPU 進行繪圖

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) 注意力機制 (Attention Mechanism)</span>  
<span style='color: #646464;'>💡 為什麼？ Transformer 模型的發明奠基於 Attention 機制。它讓模型在處理長句子時，能學會要把「目光」放在哪些關鍵字上 (例如：代名詞 He 對應的是前文中的哪個人名)，這是機器突破翻譯與理解瓶頸的技術關鍵。</span>

### Q16: 在數據清洗過程中，若數據發生嚴重偏差（例如資料都只採樣特定族群），會導致？

- (A) 電腦死機
- (B) 運算速度變慢
- <span style='color: #0066cc; font-weight: bold;'>(C) 模型結果產生歧視或偏見</span>
- (D) 資料容量變大

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 模型結果產生歧視或偏見</span>  
<span style='color: #646464;'>💡 為什麼？ Garbage in, Garbage out。如果訓練資料本身就帶有偏見 (如招募資料中男性錄取多)，AI 就會學到「男性更適合」這個錯誤邏輯，從而在應用階段產生性別等歧視性結果。</span>

- (D) 未經同意隨意收集他人生物特徵

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) 在進行數據分析前先進行去識別化處理</span>

### Q20: 在模型預測正確率（Accuracy）很高，但遺漏了關鍵少數案例（例如 99 ％正常，卻漏掉關鍵 1％病患）時，我們應關注哪一指標？

- (A) 總數據量
- (B) 模型運算次數
- <span style='color: #0066cc; font-weight: bold;'>(C) 召回率 (Recall)</span>
- (D) 設定參數

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 召回率 (Recall)</span>  
<span style='color: #646464;'>💡 為什麼？ 召回率衡量的比例是「原本該被抓出來的，模型抓出了多少」。在醫療或金融防詐場景下，「漏掉壞人」的代價非常巨大，因此 Recall 是比 Accuracy 更有意義的衡量指標。</span>

### Q21: 下列何者並非大型語言模型 (LLM) 目前主要的限制？

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(D) 無法處理純文本資料</span>  
<span style='color: #646464;'>💡 為什麼？ 文本資料是 LLM 賴以生存的核心輸入。目前 LLM 的技術瓶頸在於：無法保證 100% 正確(幻覺)、需要昂貴顯卡進行超大規模運算、以及訓練資料庫本身帶有的社會偏見。</span>
