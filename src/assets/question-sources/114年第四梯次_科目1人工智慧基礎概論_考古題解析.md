### Q1: 在人工智慧系統的決策流程中，下列哪一種情境最符合「人在迴圈上 （Human-over-the-loop）」所強調的監督機制？

- (A) AI 系統只能提供建議，人類需主動下達命令才能進行決策；
- <span style='color: #0066cc; font-weight: bold;'>(B) 人類對 AI 的運行進行日常監督，必要時可立即介入修正或干預；</span>
- (C) 人類平時不參與 AI 的運作，僅在發生異常或重大錯誤時才接手控 制；
- (D) AI 的所有判斷與行動在執行前，皆須經過人類逐一審核與批准

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) 人類對 AI 的運行進行日常監督，必要時可立即介入修正或干預；</span>  
<span style='color: #646464;'>💡 為什麼？ 「人在迴圈上 (Human-over-the-loop)」意指 AI 系統能獨立運作，但人類扮演監督者的角色，不僅監控結果，也能隨時視情況介入並覆寫 AI 的決策。這有別於「人在迴圈內 (Human-in-the-loop)」的必須由人類拍板定案。</span>

### Q2: 下列哪一種特徵工程技巧，最適合將「星期幾」和「24 小時制時間」 這兩個欄位結合，以預測通勤時間？

- (A) One-hot 編碼（One-hot encoding）；
- (B) 正規化（Normalization）；
- <span style='color: #0066cc; font-weight: bold;'>(C) 特徵交叉（Feature Cross）；</span>
- (D) 寬深模型（Wide and Deep）

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 特徵交叉（Feature Cross）；</span>  
<span style='color: #646464;'>💡 為什麼？ 「特徵交叉」是將多個獨立的類別或數值特徵合併為單一特徵的工程技術。將「星期幾」與「小時」交叉組合（例如：星期一早上8點），能幫助模型捕捉到兩個變數結合後對目標變數（通勤時間）產生的非線性交互作用。</span>

### Q3: 關於 ETL（Extract-Transform-Load） ，下列敘述何者為正確？

- (A) E 表示將資料直接儲存至目標儲存庫；
- (B) ETL 的處理順序可以自由調整為 TEL；
- (C) L 表示將目標儲存庫經過反加密處理載入資料；
- <span style='color: #0066cc; font-weight: bold;'>(D) T 包括資料的清理與排序</span>

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(D) T 包括資料的清理與排序</span>  
<span style='color: #646464;'>💡 為什麼？ ETL 分別代表萃取 (Extract)、轉換 (Transform)、載入 (Load)。其中的 T (Transform) 階段負責將原始資料進行篩選、清理、排序與格式轉換，以確保進入資料倉儲的資料品質與一致性。</span>

### Q4: 關於資料正則化 （Regularization）L1、L2 方法 ， 下列敘述何者正確？

- (A) L1 權重個數愈多，愈可以提升模型的正確率；
- (B) L2 稱為 Lasso 正則化；
- <span style='color: #0066cc; font-weight: bold;'>(C) L1 運用減少權重的絕對值來控制模型的複雜度；</span>
- (D) L2 較 L1 正則化方法會將特徵權重趨近於零

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) L1 運用減少權重的絕對值來控制模型的複雜度；</span>  
<span style='color: #646464;'>💡 為什麼？ L1 正則化 (Lasso) 透過在損失函數加入權重絕對值的懲罰項，不僅能控制模型複雜度避免過擬合，其數學特性更會迫使不重要的特徵權重收斂至 0，從而達到特徵選擇 (Feature Selection) 的效果。</span>

### Q5: 在機器學習中， 「偏差與變異權衡 （Bias-Variance Tradeoff） 」 主要用 來解決下列哪一類型的問題？

- (A) 因資料來源或收集方式限制，導致模型學習到的資訊不足；
- (B) 測試資料樣本與訓練資料高度重複，造成模型泛化能力評估失準；
- (C) 訓練資料中類別分布不均，使模型在少數類別上表現不佳；
- <span style='color: #0066cc; font-weight: bold;'>(D) 如何在模型偏差與變異之間取得平衡，以避免過度擬合或欠擬合</span>

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(D) 如何在模型偏差與變異之間取得平衡，以避免過度擬合或欠擬合</span>  
<span style='color: #646464;'>💡 為什麼？ 這是機器學習的核心難題。偏差 (Bias) 過高會導致模型無法捕捉資料規律（欠擬合）；變異 (Variance) 過高則會讓模型過度記憶訓練資料雜訊（過擬合）。權衡兩者是為了找出泛化能力最佳的模型狀態。</span>

### Q6: 在 Lasso 模型中，L1 正則化（Regularization）導致參數收斂為零的 原因為何？

- (A) L1 正則化忽略目標變數；
- (B) L1 對梯度有平滑作用；
- <span style='color: #0066cc; font-weight: bold;'>(C) L1 對大係數懲罰較強，促使稀疏解；</span>
- (D) L1 會轉換損失函數為非凸形

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) L1 對大係數懲罰較強，促使稀疏解；</span>  
<span style='color: #646464;'>💡 為什麼？ L1 正則化的懲罰項在幾何空間中呈現多邊形 (菱形) 分布。這種形狀的頂點容易與損失函數的等高線在座標軸上相交，從而讓許多影響較小的特徵權重直接變為 0，產生稀疏的解答。</span>

### Q15: 某模型使用 K-近鄰演算法（KNN）進行分類，K 設為 3。 一筆新的測 試資料輸入後，與其最接近的3 筆資料的類別如下：鄰近樣本1：類別 A；鄰近樣本 2：類別B；鄰近樣本 3：類別A。請問模型會將這筆資料 預測為哪一類別？

- <span style='color: #0066cc; font-weight: bold;'>(A) 類別 A；</span>
- (B) 類別 B；
- (C) 類別 A 與 B 各一半，無法分類；
- (D) 類別 A 或 B，視距離遠近加權而定

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(A) 類別 A；</span>  
<span style='color: #646464;'>💡 為什麼？ K近鄰演算法 (KNN) 的分類決策法則為「多數決 (Majority Voting)」。當 K=3 時，鄰近的 3 筆樣本中有 2 筆為類別 A，1 筆為類別 B，因此模型最終會預測該測試資料為數量較多的類別 A。</span>

### Q16: 某團隊想採用循環神經網路（Recurrent Neural Network, RNN）建構 長期氣候數據的預測模型，以下哪一項敘述最符合使用 RNN 可能會遇 到的挑戰？

- (A) RNN 無法處理可變長度的序列輸入，因此在實務上限制極大；
- <span style='color: #0066cc; font-weight: bold;'>(B) RNN 在長序列訓練中可能出現梯度消失，影響模型效果；</span>
- (C) RNN 無法捕捉時間上的依賴關係，因此預測準確度低；
- (D) RNN 只能用於分類任務，不能應用於時間序列預測

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) RNN 在長序列訓練中可能出現梯度消失，影響模型效果；</span>  
<span style='color: #646464;'>💡 為什麼？ 傳統 RNN 在處理長序列時，由於反向傳播過程中梯度不斷連乘，若權重小於 1，梯度會隨時間步數指數級衰減並趨近於 0（即梯度消失問題），導致模型無法回憶並學習到很久以前的關鍵時間依賴資訊。</span>

### Q17: 一間金融科技公司設計一款智慧投資系統，該系統會根據市場變化自 動決定 「買進」 、 「持有」 或 「賣出」 的行動，並根據每次交易後的盈虧 結果 ， 逐步優化下一次的投資策略 。 整個過程中 ， 系統不依賴事先標記 的資料 ， 而是根據歷次行動獲得的獎勵進行調整 。 請問此系統最可能採 用哪一種學習方法？

- <span style='color: #0066cc; font-weight: bold;'>(A) 強化式學習（Reinforcement Learning）；</span>
- (B) 監督式學習（Supervised Learning）；
- (C) 非監督式學習（Unsupervised Learning）；

- <span style='color: #0066cc; font-weight: bold;'>(C) 根據顧客的消費行為與特徵，將顧客自動劃分為數個群組，以便進 行差異化行銷；</span>
- (D) 透過大量已標註影像，訓練深度學習模型來辨識照片中的物件種類

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 根據顧客的消費行為與特徵，將顧客自動劃分為數個群組，以便進 行差異化行銷；</span>  
<span style='color: #646464;'>💡 為什麼？ 叢集 (Clustering) 屬於無「標準答案」的非監督式學習。演算法會自動根據資料間的特徵距離將相似的樣本歸納成數個群體（例如：發現「高消費低頻次」與「低消費高頻次」兩大客群）。其它選項皆具備明確的預測標籤，屬於監督式學習範疇。</span>

### Q45: 某醫院研究團隊蒐集了大量病患的 「收縮壓」 數據 ， 經檢驗後顯示此數 值大致呈現常態分布 。 在進行後續模型分析前 ， 研究人員希望妥善處理 可能存在的極端血壓數值。下列哪一種做法最為合適？

- (A) 將所有極端偏高或偏低的血壓數據直接刪除，以保留最具代表性的 病患樣本；
- (B) 使用對數轉換 （Log Transformation） ，將數據壓縮至更接近常態， 以降低極端值的影響；
- <span style='color: #0066cc; font-weight: bold;'>(C) 透過 Z 分數 （Z-score） 或標準差範圍檢測異常值 ， 並依研究需求決 定是否調整或移除；</span>
- (D) 將檢測到的離群值以 Label Encoding 編碼，轉換為序號標籤以避 免影響原始分布

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(C) 透過 Z 分數 （Z-score） 或標準差範圍檢測異常值 ， 並依研究需求決 定是否調整或移除；</span>  
<span style='color: #646464;'>💡 為什麼？ 在常態分布假設下，約 99.7% 的資料點會落在正負 3 個標準差 (Z-score = ±3) 內。據此可客觀計算出何謂離群值 (Outliers)。然而重症患者的極端數值往往是重要的醫學線索，不能盲目刪除，應在標定後交由臨床研究需求來做最終決定才最為嚴謹。</span>

### Q46: 某電商公司想預測用戶是否會購買特定商品，資料中包含多種用戶屬 性與行為特徵 。 分析師希望選出對購買結果最有預測價值的特徵 ， 以提 升模型效能。下列哪一種描述最符合監督式特徵選擇（ Supervised Feature Selection）的概念？

- (A) 根據特徵的整體分布、變異度或資訊量進行篩選，而不直接參考目 標變數；
- <span style='color: #0066cc; font-weight: bold;'>(B) 評估每個特徵與目標變數之間的相關性，選擇對預測結果貢獻最大 的特徵；</span>
- (C) 使用模型評估特徵對預測結果的重要性，並保留對目標變數影響較 大的欄位；

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) 評估每個特徵與目標變數之間的相關性，選擇對預測結果貢獻最大 的特徵；</span>  
<span style='color: #646464;'>💡 為什麼？ 「監督式」特徵選擇的核心在於利用已知的「目標標籤」。它會計算每個變數與目標變數之間的統計相關強度（如皮爾森係數、資訊增益等），藉此篩選出預測力最強的欄位；而選項 (A) 和 (D) 皆屬於忽略目標變數的非監督式降維或篩選作法。</span>

### Q47: 在訓練神經網路時 ， 為了提升模型收斂速度與穩定性 ， 避免梯度消失或 梯度爆炸，下列哪一種做法最常被使用？

- (A) 對輸入資料進行隨機旋轉或水平翻轉，以增加資料多樣性；
- <span style='color: #0066cc; font-weight: bold;'>(B) 選用 ReLU 或其變體作為隱藏層的啟動函數，以改善梯度傳播；</span>
- (C) 減少樣本量提升訓練速度；
- (D) 對目標變數或特徵進行標準化

<span style='color: #008000; font-weight: bold;'>✅ 正確解答：(B) 選用 ReLU 或其變體作為隱藏層的啟動函數，以改善梯度傳播；</span>  
<span style='color: #646464;'>💡 為什麼？ 傳統的 Sigmoid 或 Tanh 啟動函數在輸入值極端時其導數趨近於零，層層連乘後會導致深層網路「梯度消失」。ReLU (Rectified Linear Unit) 函數在正數區間的導數恆為 1，有效解決梯度在反向傳播途中的衰減問題，確保了深層網路的穩定收斂。</span>
